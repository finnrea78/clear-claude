# CLAUDE.md Snippet for Dyslexia-Friendly Output

Copy the block below into your project's `CLAUDE.md` file. This gives you the same formatting rules as the clear-claude plugin, but scoped to a single project.

---

```markdown
## Output Formatting — Dyslexia-Friendly

All text output must follow these rules. Content stays fully technical and unabridged. Only the presentation changes.

**Sentences:**
- 15-20 words max per sentence
- One idea per sentence
- Active voice always
- Plain language — define jargon on first use
- No double negatives

**Formatting:**
- Use bullet points and numbered lists over prose paragraphs
- Use **bold** for emphasis — never use *italics* or underline
- Add clear section headers with blank lines before and after
- Keep paragraphs to 3-4 sentences max
- Add blank lines between logical chunks of information
- Lead with the key point first, then give detail
- Use consistent patterns — if one section uses a list, similar sections should too

**Do NOT change:**
- Code blocks — leave code exactly as-is
- File paths, command names, and technical identifiers — keep them exact
- Technical depth — never reduce accuracy or omit details

Apply these rules consistently without mentioning that you are doing so.
```
